# RogueSpire ゲームバランス＆拡張ロードマップ

## 現状分析

本作の現在の状態を、類似タイトルの設計原則と照らし合わせて分析します。

### ✅ 既に良い点
| 要素 | 本作の実装 | 類似タイトルとの比較 |
|------|-----------|-------------------|
| インテントシステム | 敵の次の行動を事前表示 | StS と同じ「戦闘はパズル」思想 ✓ |
| エナジー制 | 3エナジー＋カードコスト | ジャンル標準の資源管理 ✓ |
| 3クラス選択 | Warrior / Silent / Mage | StS の3クラスと同等 ✓ |
| 状態異常 | 毒・脱力・脆弱・トゲ・反射 | 豊富なデバフ ✓ |
| リスク/リターンイベント | 祭壇・ミミック箱・闇商人 | トレードオフの選択肢 ✓ |

### ⚠️ 改善が必要な点（類似タイトルとの差）

| 問題 | 詳細 | 参考タイトル |
|------|------|-------------|
| **敵のスケーリングが数値のみ** | フロアが進むほどHPが増えるだけで、敵の「行動の複雑さ」が増えない | StS / Wildfrost |
| **戦闘が均一化** | 全フロアで「攻撃→防御→ターンエンド」のパターンが最適解になりがち | Monster Train |
| **カードの「ジョブ」が不明確** | 早期終結用カード vs 長期戦スケーリングカードの区別がない | StS の Jobs 理論 |
| **デッキ圧縮の動機が弱い** | カード除去のメリットが感じにくい | Balatro / StS |
| **ボスのギミック不足** | ドラゴンがただのHP高い敵。プレイヤーのエンジンを妨害する固有ギミックがない | Balatro Boss Blinds |

## 類似タイトルの核心設計パターン

### 1. Slay the Spire：「スケーリング競争」

**最大の教訓：** 敵もプレイヤーと同様にターンごとに強くなる。

- **筋力スケーリング敵**: ターンごとに ATK+2 される敵がいる → プレイヤーは「速攻で倒すか、ブロックをスケーリングさせるか」の判断を迫られる
- **Wound / Dazed の挿入**: 敵が山札にゴミカード（WOUND=使用不可/0コスト枠埋め）を混入 → プレイヤーのドロー品質を劣化させる
- **時限爆弾敵**: 数ターン後に大ダメージを放つ敵 → 速攻かデバフで対処するしかない

### 2. Balatro：「ボスがシナジーを壊す」

**最大の教訓：** ボスはただ強いだけでなく、プレイヤーの戦略そのものを否定するギミックを持つ。

- **The Pillar**: このラウンドで既にプレイしたカードを全て無効化
- **The Plant**: ジョーカー（レリック相当）の効果を無効化
- **The Needle**: 1回だけしか手を出せない

→ プレイヤーは「1つの最強コンボ」に依存できず、複数の勝ち筋を用意する必要がある。

### 3. Wildfrost：「ウェーブと退避」

**最大の教訓：** 敵はグループ（ウェーブ）で出現し、優先順位を判断する必要がある。

- **ヒーラー敵**: 他の敵を毎ターン回復する → 最優先で倒す必要がある
- **タンク敵**: 高HPで前線を張り、背後の危険な敵を守る
- **Shroom（毒キノコ）**: 毒スタックを付与し続ける → 放置するとダメージが指数的に増大

### 4. Monster Train：「階層防衛」

**最大の教訓：** 敵の行動パターンに「貫通（Trample）」「範囲攻撃（Sweep）」などの特性を持たせる。

- **Trample**: ダメージがオーバーキルした場合、余剰分が次のユニットに流れる
- **Sweep**: 同じ行の全ユニットにダメージを与える
- **Stealth**: 直接攻撃のターゲットにできない → スペルや全体攻撃でしか倒せない

## 本作への具体的拡張ロードマップ（優先度順）

### 🔴 最優先：敵の行動複雑化 + ボスギミック

#### A. 敵の「特性（Trait）」システム導入
各敵に固有の戦闘開始時の特性を持たせる。

| 敵 | 新特性 | 効果 | デザイン意図 |
|----|--------|------|-------------|
| GOBLIN (F1) | **素早い** | 初手は必ず攻撃（変更なし） | チュートリアル敵。シンプル |
| SLIME (F2) | **分裂** | HP50%以下で「ミニスライム」(HP10)を召喚。本体+子で攻撃 | 速攻の動機付け |
| ORC (F3) | **怒り** | 3ターン目以降、毎ターンATK+2ずつスケーリング | 速攻 vs 防御のジレンマ |
| MAGE (F4) | **詠唱** | 毎ターン「BURNカード」(使用不可/手札圧迫)をプレイヤーの山札に追加 | デッキ汚染への対処を要求 |
| DRAGON (F5) | **2フェーズ制** | HP50%でフェーズ移行演出。フェーズ2ではプレイヤーの最大エナジーを1減少させるオーラ | エンジン妨害ボス |

#### B. ドラゴン戦のボスギミック追加
Balatroの「Boss Blind」に倣い、ドラゴンに固有ギミックを追加：
- フェーズ1: 通常攻撃パターン（現在と同じ）
- **フェーズ移行演出**: HP50%で画面全体フラッシュ + テキスト「THE DRAGON ROARS!」
- フェーズ2: 以下の3つから毎周ランダム1つ選択
  - 「DRAGON CURSE」: プレイヤーの最大エナジーが1減少
  - 「FLAME AURA」: プレイヤーのブロックを毎ターン半分に減少
  - 「ANCIENT POWER」: ドラゴンが毎ターンATK+3スケーリング

### 🟡 高優先：カード経済の改善

#### C. 「Wound（傷）」カードの追加
StSの設計哲学に直接基づく、敵がプレイヤーの山札を汚染するメカニズム。

```
wound: { name: "WOUND", cost: 9, type: "CURS", value: 0, desc: "UNPLAYABLE / NO EFFECT", rare: "COMMON" }
burn:  { name: "BURN",  cost: 9, type: "CURS", value: 0, desc: "UNPLAYABLE / T-END TAKE 1 DMG", rare: "COMMON" }
```

- MAGE敵がBURNをプレイヤーに挿入
- DRAGON敵がWOUNDをプレイヤーに挿入
- TRUE GRITのExhaust機能がこれらの除去手段になる → 廃棄メカニクスの価値が上昇

#### D. カードプール設計の明確化
StSの「ジョブ理論」に基づき、各カードに明確な役割を割り当て：

| ジョブ | 説明 | 該当カード例 |
|--------|------|-------------|
| **Frontload（速攻）** | 短期戦で即座に効果を発揮 | STRIKE, BASH, CARNAGE |
| **Scaling（成長）** | 長期戦で威力が増大 | RUPTURES, ACCURACY, INF BLADES |
| **Mitigation（防御）** | ダメージを軽減 | DEFEND, FLAME BARRIER, G-ARMOR |
| **Utility（汎用）** | ドロー・エナジー操作 | FOCUS, ADRENALINE, OFFERING |
| **Curse Counter（呪い対策）** | 廃棄でデッキを浄化 | TRUE GRIT, SECOND WIND |

### 🟢 中優先：リプレイ性の向上

#### E. ORCエリートの「怒りスケーリング」
```
// ORCの行動パターン改善
if (eTurnCount >= 3) {
  // 3ターン目以降、蓄積される怒りバフ
  let rageStacks = eTurnCount - 2;
  eIntent.value += rageStacks * 2;
}
```
→ 長引くほど敵が強くなる = 速攻or十分なブロックが必要

#### F. ミニボス: SLIMEの分裂
HP50%以下で「ミニスライム(HP10, ATK4)」を追加敵として召喚。
- 追加で2体まで出現
- 本体を倒しても、ミニスライムが生きていれば戦闘継続
→ 範囲ダメージの価値が上がる

#### G. 新レリック追加（レリックの選択肢を増やす）
| レリック | 効果 | コスト |

| BURNING BLOOD | 戦闘終了時HP+6回復 | 50G |
| SNECKO SKULL | 毒を付与するたび+1追加 | 45G |

## バランス調整の数値ガイドライン

類似タイトルの分析から導出した、本作のバランス基準値：

| 指標 | 目標値 | 根拠 |
|------|--------|------|
| F1戦闘ターン数 | 3-4ターン | チュートリアル的。サクサク終わるべき |
| F3戦闘ターン数 | 5-7ターン | エリート戦。緊張感のある長さ |
| F5ボス戦ターン数 | 8-12ターン | 全リソースを使い切る死闘 |
| 1戦あたりの被ダメージ | 平均HP20-30%消費 | 回復リソースとの兼ね合い |
| カード報酬の「ピック率」 | 各カード10-30%が理想 | 0%や100%のカードは調整が必要 |
| クリア率 | 初見30-40% / 熟練60-70% | 難しすぎず、簡単すぎず |
